# 자동 문서 작성 시스템

항상 한국어로 응답한다.

## 시작

스킬이 트리거되면 사용자에게 묻는다:

> 어떤 주제의 문서를 작성하고 싶으신가요?

주제를 받으면 아래 7단계를 순서대로 자동 실행한다.

---

## 7단계 실행 흐름

### Step 1: 소제목 & 용어 수집

주제를 분석해 문서에 필요한 소제목 목록과 핵심 용어를 정리한다. 사용자에게 구성안을 보여주고 계속 진행할지 확인한다. 변경 요청이 있으면 반영 후 Step 3으로 넘어간다.

### Step 2: 1차 검토 (수준 조정)

Step 1의 구성안에서 대상 독자 수준에 비해 과도하게 깊거나 범위를 벗어난 항목을 제거하거나 조정한다. 조정 내역을 간략히 알린다.

### Step 3: 초안 작성

확정된 구성을 바탕으로 서술식 초안을 작성한다.

**작성 원칙:**
- 단순 용어 정의가 아닌 이야기하듯 설명
- 개념이 왜 필요한지부터 시작
- 문제 → 해결책 흐름 활용 (주제에 맞게)
- 언리얼 엔진 / C++ 예시 우선 사용
- 코드 예시는 실제 컴파일 가능한 수준으로 작성

### Step 4: 2차 정제

초안에서 과도하게 깊은 내용, 반복, 불필요한 군더더기를 제거한다. 독자 수준에 맞게 조정하고 흐름을 매끄럽게 다듬는다.

### Step 5: 마크다운 문법 검수

**검사 항목:** 코드 블록(```), 헤더 레벨, 링크·이미지 문법, 표 형식, 강조 문법(**bold**, *italic*), 목록 들여쓰기, 특수 문자 이스케이핑

잘못된 문법 수정 및 형식 통일 후 결과를 보고한다.

### Step 6: 코드 예시 검증

**검사 항목:** 문법 오류, 변수명 오타, 함수 시그니처, 타입 불일치, 메모리 관리 실수, 언리얼·C++ 특화 내용, 주석과 코드 일관성

잘못된 코드 수정 및 설명 보완 후 결과를 보고한다.

### Step 7: 환각 & 창작 내용 검증

**검사 항목:** 존재하지 않는 함수·클래스·API, 잘못된 버전 정보, 부정확한 역사적 정보, 존재하지 않는 언리얼 기능, 부정확한 성능 수치

불확실한 내용 제거 후 검증 가능한 사실만 남긴다. 결과를 보고한다.

## 파일 저장

완성된 문서는 주제에 맞는 `D:\DoveObs\` 하위 볼트에 저장한다.

### 볼트 선택 기준

| 볼트 | 해당 주제 |
|------|-----------|
| `D:\DoveObs\Unreal\` | 언리얼 엔진 (블루프린트, GAS, 애니메이션, 렌더링, 모듈, 델리게이트 등) |
| `D:\DoveObs\Niagara\` | 나이아가라 파티클 시스템 |
| `D:\DoveObs\Optimization\` | 성능 최적화 (메모리, CPU, GPU, 프로파일링 등) |
| `D:\DoveObs\3DCG_Channel\` | 3D 툴 (Blender, Maya, 3ds Max, ZBrush, Substance Painter 등) |
| `D:\DoveObs\Tools\` | 개발 도구 및 워크플로우 |
| `D:\DoveObs\Global\` | 위 볼트에 해당하지 않는 범용 주제 (C++ 일반, 알고리즘, 자료구조 등) |

주제를 받으면 먼저 어느 볼트에 저장할지 판단하고 사용자에게 알린다. 볼트 결정이 애매한 경우 사용자에게 확인한다.

파일명은 주제를 영어로 간결하게 표현하고, 앞에 `(Claude)` 접두어를 붙인다.

예: `D:\DoveObs\Unreal\(Claude)UE SmartPointer.md`, `D:\DoveObs\Global\(Claude)BinarySearch.md`
