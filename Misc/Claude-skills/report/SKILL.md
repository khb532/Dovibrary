일일 작업 레포트를 자동으로 생성한다. 사용자 확인 없이 즉시 실행한다.

## 저장 경로

고정 경로: `D:\DoveObs\Reports\Daily_Report_{YYYY-MM-DD}.md`

경로가 없으면 자동 생성:
```bash
mkdir -p "/d/DoveObs/Reports"
```

오늘 레포트가 이미 있으면 현재 상태로 덮어쓴다.

## 정보 수집 순서

### 1. 프로젝트 정보 (자동 추론 + 필요시 질문)

- **프로젝트명**: 현재 작업 디렉토리명 또는 git repo명으로 추론
- **현재 단계**: TODO/문서에서 "Alpha", "Beta", "MVP" 등 키워드 추출, 없으면 컨텍스트 추론
- **목표**: `Document/plan/*.md` 또는 대화 컨텍스트에서 추출, 없으면 사용자에게 질문
- **작업 브랜치**: `git branch --show-current` 자동 감지

### 2. 작업 내용 (우선순위 순)

**TODO가 있는 경우**: TodoRead/TaskList 도구로 현재 상태 확인
- completed → 완료된 작업
- in_progress → 진행 중 작업
- pending → 미완료 작업

**TODO가 없는 경우**: Git으로 전환
```bash
git diff HEAD      # 변경사항
git status         # 새 파일(untracked)
```
파일 경로에서 작업 성격 추론 (예: `*.h` → 헤더 작성, `*.md` → 문서 작성)

### 3. 학습 내용 (자동 추론)

현재 대화 컨텍스트에서 학습한 개념 3~5개 추출:
- 새로 배운 클래스/함수/패턴
- 질문-답변에서 등장한 핵심 개념
- 작성한 코드에서 사용된 주요 기법

## 레포트 템플릿

```markdown

- **전체 프로젝트**: {프로젝트 설명}
- **현재 단계**: {단계명}
- **{단계} 목표**: {목표 설명}
- **작업 브랜치**: {브랜치명}

---

## ✅ 완료된 작업

**생성된 파일**:
{완료된 TODO의 결과물 파일 리스트}

**작업 중 파일** (있는 경우):
{in_progress 상태의 TODO 파일 + 진행 상황}

---

## 📊 전체 TODO 현황

**TODO 있는 경우**: ({완료수}/{전체수} 완료)

1. ✅ **{완료된 TODO 제목}**
2. ⏳ **{진행중 TODO 제목}** (다음 작업)
   - [ ] {세부 작업 1}
   - [ ] {세부 작업 2}
3. ⏳ **{미완료 TODO 제목}**

**TODO 없는 경우**:
⚠️ TODO를 사용하지 않음 (Git 변경사항 기반 레포트)

---

## 💡 학습 내용

{오늘 학습한 주요 개념 3~5개를 한 줄씩 요약}

---

**작성일**: {YYYY년 MM월 DD일}
```

## 완료 메시지

```
✅ 일일 레포트가 생성되었습니다!

📄 파일: D:\DoveObs\Reports\Daily_Report_{YYYY-MM-DD}.md
📊 TODO: {완료수}/{전체수} 완료
```

## 오류 처리

- **TODO도 없고 Git 변경사항도 없는 경우**: 레포트 생성 불가 안내
- **경로 없음**: 자동 mkdir
- **Git 없는 프로젝트**: 브랜치 항목 생략

## 주의사항

- 항상 한국어로 작성
- 각 섹션은 핵심만 담아 간결하게
- 가능한 모든 정보는 자동 수집, 대화 최소화
- 범용 템플릿 — Unreal Engine 외 모든 프로젝트에 동일하게 적용
